# 实现统一的批量下载和文件夹下载功能

## 功能分析
将批量下载和文件夹下载功能合并为一个统一的API端点，支持：
1. **批量下载**：选择多个文件后一次性下载
2. **文件夹下载**：下载整个文件夹及其所有内容
3. **混合下载**：同时下载多个文件和文件夹

## 实现方案

### 后端实现
1. **添加统一的下载API端点** (`/api/files/<client_db_id>/download`)
   - 支持多种参数形式：
     - `path`：单个文件路径
     - `paths`：文件路径数组（批量下载）
     - `folder`：文件夹路径（文件夹下载）
   - 处理逻辑：
     - 如果提供了`paths`数组，批量下载所有文件
     - 如果提供了`folder`，递归下载整个文件夹
     - 如果同时提供了`paths`和`folder`，下载所有内容
     - 如果只提供了`path`，保持原有单个文件下载功能
   - 创建临时zip文件，将所有文件添加到压缩包
   - 保持文件的原始目录结构
   - 返回zip文件下载响应
   - 下载完成后清理临时文件

2. **优化现有文件重建函数**
   - 确保`reconstruct_file_version`函数支持从存储块中正确重建文件
   - 处理大文件和增量备份的情况

### 前端实现
1. **添加文件多选功能**
   - 支持Ctrl+点击多选文件
   - 支持Shift+点击连续选择
   - 添加全选按钮
   - 显示已选择文件数量

2. **添加批量操作工具栏**
   - 批量下载按钮（当选择了多个文件时显示）
   - 批量操作菜单

3. **添加文件夹下载功能**
   - 在文件夹右键菜单中添加"下载"选项
   - 在文件夹操作按钮中添加下载功能

4. **优化下载体验**
   - 添加下载进度提示
   - 处理大文件下载的超时问题
   - 添加下载完成提示

### 技术要点
- 使用`zipfile`库创建压缩包
- 临时文件管理：使用`tempfile`模块创建临时文件
- 前端多选实现：基于JavaScript的事件处理
- 递归文件夹遍历：构建虚拟目录树的递归遍历
- 性能优化：对于大文件夹，考虑流式下载和内存使用

## 实施步骤
1. 修改`web/app.py`添加统一的下载API端点
2. 修改`web/templates/files.html`添加前端多选和批量操作功能
3. 测试批量下载和文件夹下载功能
4. 优化性能和用户体验

## 预期效果
- 用户可以选择多个文件并点击批量下载按钮一次性下载
- 用户可以右键点击文件夹并选择"下载"来下载整个文件夹
- 下载的文件会被打包成zip格式，保持原有的目录结构
- 提供良好的用户反馈和错误处理
