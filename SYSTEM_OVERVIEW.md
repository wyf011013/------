- URL: http://localhost:5000
- 账号: admin / admin123

## 🔧 核心模块说明

### 1. 自定义TCP协议 (`client/protocol.py`)

#### 协议格式
```
+--------+--------+--------+--------+--------+
| 魔数   | 版本   | 类型   | 长度   | 数据   |
| (4B)   | (1B)   | (1B)   | (4B)   | (N B)  |
+--------+--------+--------+--------+--------+
```

#### 消息类型
```python
class MessageType(IntEnum):
    CLIENT_REGISTER = 0x01      # 客户端注册
    CLIENT_HEARTBEAT = 0x02     # 心跳包
    BACKUP_CONFIG = 0x03        # 备份配置下发
    FILE_CHANGE = 0x04          # 文件变化通知
    FILE_DATA = 0x05            # 文件数据传输
    FILE_CHUNK = 0x06           # 文件分块传输
    BACKUP_STATUS = 0x07        # 备份状态
    ERROR = 0x08                # 错误消息
```

### 2. 文件监控 (`client/file_monitor.py`)

#### 功能特性
- **实时监控**: 使用watchdog库监控文件系统事件
- **防抖处理**: 避免频繁写入触发多次备份
- **批量处理**: 合并多个文件事件，提高效率
- **排除模式**: 支持通配符排除不需要备份的文件

#### 监控事件
- 文件创建
- 文件修改
- 文件删除
- 文件移动

### 3. 存储管理 (`server/storage_manager.py`)

#### 存储策略
- **块级存储**: 文件分块存储，每块64KB
- **数据去重**: 相同哈希的块只存储一次
- **数据压缩**: 启用zlib压缩，减少存储占用
- **版本管理**: 每个版本记录元数据和块引用

#### 目录结构
```
backup_storage/
├── clients/                    # 按客户端分类
│   ├── {client_id}/
│   │   ├── files/              # 文件数据
│   │   │   ├── {file_hash}.blk # 数据块
│   │   │   └── ...
│   │   ├── versions/           # 版本信息
│   │   │   ├── {version_id}.json
│   │   │   └── ...
│   │   └── index.json          # 客户端索引
├── shared_blocks/              # 全局去重块
│   └── {hash}.blk
└── system/                     # 系统文件
    ├── logs/
    └── config/
```

### 4. 数据库设计 (`sql/init_database.sql`)

#### 核心表
- **clients**: 客户端信息表
- **backup_configs**: 备份配置表
- **file_versions**: 文件版本表
- **users**: 用户表
- **system_logs**: 系统日志表

#### 视图
- **client_statistics**: 客户端统计视图
- **storage_statistics**: 存储统计视图

### 1. 存储优化
- ✅ **块级去重**: 相同数据块只存一份
- ✅ **数据压缩**: zlib压缩减少存储占用
- ✅ **定期清理**: 自动清理旧版本文件

### 2. 网络优化
- ✅ **增量传输**: 只传输变化的文件部分
